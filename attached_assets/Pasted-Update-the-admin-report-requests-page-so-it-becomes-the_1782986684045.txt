Update the admin-report-requests page so it becomes the fulfilment centre for paid Business Reputation Reports.

Current flow:
1. Business owner clicks Unlock Full Report
2. They enter details in the request form
3. Request is saved in the report_requests database table as pending_payment
4. They go to Stripe and pay

New admin workflow required:
In /admin-report-requests, each request row should show:
- customer name
- email
- phone
- business name
- business address
- business website / Google profile link
- payment status
- report status
- request ID
- date created

Add these buttons per request:
1. Copy email
2. Copy business details
3. Mark as paid
4. Generate report
5. Download report
6. Mark report as sent

Button rules:
- If payment status is pending_payment, show:
  “Mark as paid”
  “Generate report” should be disabled or hidden.

- If payment status is paid and report status is not generated, show:
  “Generate report”

- If report status is generated, show:
  “Download report”
  “Mark report as sent”

- If report status is sent, show badge:
  “Sent”

Status values:
payment_status:
- pending_payment
- paid
- refunded
- cancelled

report_status:
- pending
- generating
- generated
- sent
- failed

Database update:
Add columns if missing:
- payment_status
- report_status
- report_pdf_url
- report_generated_at
- report_sent_at
- report_json
- stripe_payment_reference optional

Generate report flow:
When admin clicks “Generate report”:
1. Change report_status to generating
2. Use the saved request details:
   - business name
   - business address
   - business link
3. Fetch current review data again for that business.
4. Fetch available review text/snippets from Google, Yelp and TripAdvisor where available.
5. Send the data to OpenAI.
6. Generate the full AI Business Reputation Report.
7. Save the report JSON in report_json.
8. Generate a PDF file.
9. Save the PDF URL in report_pdf_url.
10. Change report_status to generated.
11. Show “Download report”.

Backend routes:
Add:

POST /api/admin/report-requests/:id/mark-paid
Requires x-admin-token

POST /api/admin/report-requests/:id/generate-report
Requires x-admin-token

POST /api/admin/report-requests/:id/mark-sent
Requires x-admin-token

GET /api/admin/report-requests/:id/download
Requires x-admin-token

Generate report endpoint:
POST /api/admin/report-requests/:id/generate-report

- Load the request from Postgres

- Create PDF
